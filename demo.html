Tested against the triangle images from
mezzoblue - IE8 Still Failing PNG Alpha

Without jquery.iepngfix

1. Transparent PNGs without CSS opacity set

Should render correctly in IE8, but have rendering issues in IE5/6/7.

2. Transparent PNGs with CSS opacity set to 0.99

Should display black lines and artifacting in IE5/6/7/8.

3. Transparent PNGs with jquery.cycle

Should display black lines and artifacting during fade transitions in IE5/6/7/8.

With jquery.iepngfix

1. Transparent PNGs without CSS opacity set

Should render correctly in all IE browsers.

2. Transparent PNGs with CSS opacity set to 0.99

Should render correctly in all IE browsers.

3. Transparent PNGs with jquery.cycle

Should render correctly in all IE browsers. Note that you must use cleartypeNoBg: true as an option to jquery.cycle's cycle(), and either directly wrap your images in container elements or pass divWrapImages: true to jquery.iepngfix's fixPNG()
